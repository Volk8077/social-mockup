Facebook ส่งคอมเมนต์มาให้เราอัตโนมัติ

เมื่อมีคนคอมเมนต์ที่เพจของลูกค้า — Facebook จะส่งสำเนาคอมเมนต์นั้นมาที่ระบบของเราภายในไม่กี่วินาที โดยไม่ต้องไปนั่งเฝ้า

กำลังเกิดอะไรขึ้น

เพจของลูกค้า (Company B) ลงทะเบียนกับ App ของเราไว้ตั้งแต่วันแรก ทำให้ Facebook รู้ว่าต้องส่ง notification มาที่ไหนเมื่อมีคอมเมนต์ใหม่ นี่คือ Meta Graph API — เป็นระบบที่ Facebook เปิดให้ developer ดึงข้อมูลและตอบกลับได้
เปรียบเทียบ: เหมือนตั้งระบบ forward email ที่เพจ — มีคอมเมนต์ใหม่ที่ไหน ระบบของเราจะรู้ทันที ไม่ต้องคอยเปิดดู

Make.com รับคอมเมนต์ — แล้วเริ่มงาน

Make คือ "เลขาฯ ดิจิทัล" ของระบบ ทำหน้าที่รับคอมเมนต์ที่ Facebook ส่งมา แล้วเริ่ม workflow ทันที — เช็คว่าเป็นคอมเมนต์ของลูกค้ารายไหน เตรียมข้อมูลให้ AI

กำลังเกิดอะไรขึ้น

Make.com เป็น เครื่องมือเชื่อมแอป เราสร้าง "scenario" คือลำดับการทำงานที่อัตโนมัติ — เมื่อ Facebook ส่งคอมเมนต์มา Make จะค้นหาใน Airtable ว่าคอมเมนต์นี้ของเพจไหน → ดึง config ของลูกค้ารายนั้น (โทนแบรนด์, KB, line_id) มาเตรียมไว้
เปรียบเทียบ: เหมือนเลขาที่รับโทรศัพท์ — รับสายแล้วเช็คทันทีว่าใครโทรมา ลูกค้ารายไหน ของบริษัทอะไร แล้วดึงไฟล์งานของลูกค้ารายนั้นมาวางไว้ก่อน

Airtable — คลังข้อมูลของทุกลูกค้า

Airtable เก็บทุกอย่างที่ระบบต้องรู้: ใครคือลูกค้ารายไหน, FAQ ของแต่ละแบรนด์, ทุกคอมเมนต์ที่เคยเข้ามา ทำให้ AI มีข้อมูลที่ถูกต้องตอบลูกค้าได้

กำลังเกิดอะไรขึ้น

ในขั้นตอนนี้ Make ดึง "Knowledge Base" ของลูกค้ารายนั้นจาก Airtable — เป็นรายการ FAQ ที่ทีม Company A เตรียมไว้: ราคาสินค้า, นโยบายส่ง, เวลาเปิดร้าน, ขนาดที่มี ฯลฯ ส่งให้ Claude ใช้อ้างอิงตอนตอบ
เปรียบเทียบ: เหมือน Excel ที่ฉลาดขึ้น — แทนที่จะมีหลายชีท Airtable เป็น ฐานข้อมูล ที่เชื่อมโยงระหว่างตารางได้ และเรียกข้อมูลผ่าน API ได้ ลูกค้าทุกรายอยู่ในตารางเดียวกัน — เพิ่มลูกค้าใหม่คือเพิ่มแถวใหม่

Claude API — สมองของระบบ

Claude อ่านคอมเมนต์ของลูกค้า + Knowledge Base ของแบรนด์ + คำสั่งจากเรา แล้วเขียนคำตอบในเสียงของแบรนด์นั้นๆ — ไม่ใช่แค่ generic "Hello, thank you" แต่ตรงประเด็น เป็นธรรมชาติ ในโทนที่ลูกค้าตั้งไว้

กำลังเกิดอะไรขึ้น

เราส่ง prompt ไปให้ Claude (ผ่าน API) ประกอบด้วย: (1) โทนแบรนด์ — สุภาพ, เป็นกันเอง, ใช้คำว่า "ค่ะ"/"ครับ" หรือไม่ (2) KB items ที่เกี่ยวข้อง (3) ข้อความคอมเมนต์ของลูกค้า · Claude อ่านแล้วคืน JSON: ระดับความเสี่ยง + ฉบับร่างคำตอบ + เหตุผลที่แบ่งระดับนั้น
เปรียบเทียบ: เหมือนพนักงาน customer service มือใหม่ที่ขยันมาก — อ่านคู่มือบริษัทเสมอก่อนตอบ ยึดโทนแบรนด์ และที่สำคัญที่สุด: รู้ตัวว่าเรื่องไหนเกินขอบเขตของตัวเอง ขอส่งให้หัวหน้า

Claude แยกความเสี่ยง — ส่งคำตอบกลับมา

ไม่ใช่ทุกคอมเมนต์ที่ AI ควรตอบเอง บางอันต้องให้คนตัดสินใจ Claude แบ่ง 3 ระดับเพื่อให้ระบบรู้ต้องทำอะไรต่อ — auto-send, รออนุมัติ, หรือส่งให้ลูกค้าจัดการเอง

กำลังเกิดอะไรขึ้น

พร้อมกับ draft คำตอบ Claude คืน "risk tier" ออกมาด้วย — เป็น label ง่ายๆ ที่ Make ใช้เลือกเส้นทางต่อ ลด cognitive load ของลูกค้าเรา (Company B) — เห็นเฉพาะที่จำเป็นต้องเห็น ไม่ต้องเปิดอ่านทุกคอมเมนต์
เปรียบเทียบ: เหมือนเลขาที่กรองอีเมล — ส่วนใหญ่ตอบให้เลย, บางอันส่งให้คุณดูก่อนตอบ, ไม่กี่อันที่บอกว่า "อันนี้คุณต้องโทรหาเอง"
